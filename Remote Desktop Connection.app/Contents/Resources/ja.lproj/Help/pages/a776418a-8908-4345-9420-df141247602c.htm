Windows ベースのコンピューターに接続できない

すべて表示

原因: Windows ベースのコンピューターでリモート接続が許可されていません。

対処方法: 接続先のコンピューターのリモート デスクトップ設定をオンにします。

Windows 7 のリモート デスクトップ設定をオンにする

重要

リモート デスクトップの設定を変更するには、Windows ベースのコンピューターの管理者権限が必要です。

  1. Windows ベースのコンピューターで、 [スタート] ボタン () をクリックし、 [コントロール パネル] をクリックします。

  2. [システムとセキュリティ] をクリックし、 [システム] で [リモート アクセスの許可] をクリックします。

  3. [リモート デスクトップ] で、必要なオプションをクリックします。

    管理者のパスワードを求めるメッセージが表示されたらパスワードを入力し、その後の確認メッセージに従って、もう一度パスワードを入力します。

    メモ  ネットワーク レベル認証によって、Mac からリモート接続を開始する際に Windows のログイン情報の入力が必要になるため、セキュリティが向上します。ネットワーク レベル認証の詳細については、Windows ヘルプを参照してください。

  4. [ユーザーの選択] をクリックします。

  5. 必要に応じて、次の操作を行います。

    目的

    操作

    アクセス許可を確認する

    自分のユーザー アカウントがボックスの一覧に表示されていることを確認します。コンピューターの管理者のアカウントは自動的に一覧に表示されます。

    他のユーザー アカウントを追加する

    [追加] をクリックし、ボックス内に、追加するユーザー アカウント名を入力します。

対処方法: Windows ファイアウォールでリモート デスクトップに対する例外が許可されていることを確認します。

Windows 7 を実行しているコンピューターで Windows ファイアウォールの例外を許可する

重要

ファイアウォールの設定を変更するには、Windows ベースのコンピューターの管理者権限が必要です。

  1. Windows ベースのコンピューターで、 [スタート] ボタン () をクリックし、 [コントロール パネル] をクリックします。

  2. [システムとセキュリティ] をクリックし、 [Windows ファイアウォール] で [Windows ファイアウォールによるプログラムの許可] をクリックします。

  3. [許可されたプログラムおよび機能] ボックスの一覧で、 [リモート デスクトップ] チェック ボックスがオンになっていることを確認します。

Windows Vista を実行しているコンピューターで Windows ファイアウォールの例外を許可する

重要

Windows XP を実行しているコンピューターで Windows ファイアウォールの例外を許可する

重要

ファイアウォールの設定を変更するには、Windows ベースのコンピューターの管理者権限が必要です。

  1. Windows ベースのコンピューターで、 [スタート] ボタンをクリックし、 [コントロール パネル] をクリックします。

    クラシック表示の場合は、ナビゲーション ウインドウの [カテゴリの表示に切り替える] をクリックします。

  2. [セキュリティ センター] をクリックし、 [セキュリティの設定の管理] の [Windows ファイアウォール] をクリックします。

  3. [例外を許可しない] チェック ボックスがオフになっていることを確認します。

  4. [例外] タブをクリックし、 [リモート デスクトップ] チェック ボックスがオンになっていることを確認します。

原因: Mac コンピューターがネットワークに接続できません。

対処方法: Windows ベースのコンピューターの電源がオンで、ネットワークに接続されていることを確認します。

ネットワークの問題が原因で、リモート接続に失敗することがあります。たとえば、リモート ネットワークがダウンしている場合や、Windows ベースのコンピューターのイーサネット ケーブルが差し込まれていない場合には、リモート接続に失敗します。

原因: ネットワーク ポリシーによって接続がブロックされています。

対処方法: 組織のセキュリティ ポリシーについてネットワーク管理者に問い合わせてください。

セキュリティ ポリシーは組織によって異なります。たとえば、ネットワーク トラフィック保護のために IPsec を使用しているネットワークで Windows ベースのコンピューターに接続するには、その Windows ベースのコンピューターについて境界の例外が必要になります。

原因: Windows ベースのコンピューターが見つかりません。

対処方法: Mac のネットワーク設定が、使用している接続の種類に合わせて正しく設定されていることを確認します。

Mac のイーサネットまたはワイヤレス接続の設定は、接続先のネットワークに合わせて設定する必要があります。

対処方法: 正しいコンピューター名が入力されていることを確認します。

Windows 7 でコンピューター名を確認する

  1. Windows ベースのコンピューターで、 [スタート] ボタン () をクリックし、 [コントロール パネル] をクリックします。

  2. [システムとセキュリティ] をクリックし、 [システム] で [コンピューターの名前の参照] をクリックします。

  3. [コンピューター名、ドメインおよびワークグループの設定] に表示されているコンピューター名を控えておきます。Windows ベースのコンピューターがドメインのメンバーの場合はフル コンピューター名を控えておきます。

    この名前は、Mac で リモート デスクトップ接続 を起動したときに入力します。

Windows Vista でコンピューター名を確認する

  1. Windows ベースのコンピューターで、 [スタート] ボタン () をクリックし、 [コントロール パネル] をクリックします。

    クラシック表示の場合は、ナビゲーション ウインドウの [コントロール パネル ホーム] をクリックします。

  2. [システムとメンテナンス] をクリックし、 [システム] をクリックします。

  3. [コンピューター名、ドメインおよびワークグループの設定] に表示されているコンピューター名を控えておきます。Windows ベースのコンピューターがドメインのメンバーの場合はフル コンピューター名を控えておきます。

    この名前は、Mac で リモート デスクトップ接続 を起動したときに入力します。

Windows XP でコンピューター名を確認する

  1. Windows ベースのコンピューターで、 [スタート] ボタンをクリックし、 [マイ コンピューター] を右クリックして [プロパティ] をクリックします。

  2. [コンピューター名] タブに表示されているフル コンピューター名を控えておきます。

    この名前は、Mac で リモート デスクトップ接続 を起動したときに入力します。

対処方法: コンピューター名を使用して接続できない場合は、コンピューターの IP アドレスを使用します。

  1. Windows ベースのコンピューターで、 [スタート] 、 [すべてのプログラム] 、 [アクセサリ] 、 [コマンド プロンプト] の順にクリックします。

  2. コマンド プロンプトで「ipconfig」と入力し、Enter キーを押します。

  3. "Ethernet adapter ローカル エリア接続" の下に表示されるコンピューターの IP アドレスを控えておきます。

    Mac で リモート デスクトップ接続 を起動したときに、コンピューター名の代わりに IP アドレスを入力できます。Windows Vista または Windows 7 を実行しているコンピューターでは、IP アドレスは IPv4 アドレスとして表示されます。

対処方法: Mac のインターネット接続が有効であることを確認します。

  • Web ブラウザー (Safari など) を開き、Web ページを表示できることを確認します。

原因: Windows ベースのコンピューターの識別情報を確認できません。

対処方法: リモート デスクトップ接続 のセキュリティの環境設定が Windows ベースのコンピューターで実行されている Windows のバージョンと互換性があることを確認します。

Windows ベースのコンピューターで、Windows XP 以前、または Windows Server 2003 Service Pack 1 (SP1) 以前のバージョンが実行されている場合、確認のための識別情報を取得することができません。 リモート コンピューターがこれらの Windows のバージョンのいずれかを実行していて、Mac のセキュリティ環境設定で接続時に認証を要求するように設定されている場合、リモート接続は失敗します。

  1. [RDC] メニューの [環境設定] をクリックし、 [セキュリティ] をクリックします。

  2. [認証が失敗した場合は接続しない] がオフになっていることを確認します。

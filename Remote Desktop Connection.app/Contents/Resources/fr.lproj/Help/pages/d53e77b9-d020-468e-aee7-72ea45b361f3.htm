Options d'authentification

Tout afficher

L'authentification de l'ordinateur distant vérifie que vous vous connectez à l'ordinateur Windows approprié. Trois options d'authentification sont disponibles.

  • Toujours se connecter, même en cas d'échec de l'authentification

    Connexion Bureau à Distance se connecte même s'il ne peut pas vérifier l'identité de l'ordinateur Windows, Il s'agit de l'option d'authentification la moins sécurisée.

    Remarque  Si un ordinateur Windows exécute Windows XP ou une version antérieure, ou Windows Server 2003 Service Pack 1 (SP1) ou une version antérieure, il ne peut pas fournir son identité pour vérification. Si vous savez que l'ordinateur Windows exécute un système d'exploitation antérieur, vous pouvez éviter les avertissements d'authentification en activant l'option Toujours se connecter, même en cas d'échec de l'authentification.

  • M'avertir en cas d'échec de l'authentification (paramètre par défaut)

    Si Connexion Bureau à Distance ne peut pas vérifier l'identité de l'ordinateur Windows, vous en êtes averti afin de pouvoir décider de continuer ou non avec la connexion. Cette option d'authentification améliore la sécurité par rapport à l'option Toujours se connecter, même en cas d'échec de l'authentification.

  • Ne pas se connecter en cas d'échec de l'authentification

    Si Connexion Bureau à Distance ne peut pas vérifier l'identité de l'ordinateur Windows, vous ne pouvez pas vous connecter. Cette option d'authentification utilise des fonctionnalités de sécurité étendues lorsque vous vous connectez à un ordinateur exécutant Windows Vista. Elle améliore la sécurité par rapport aux deux autres options d'authentification.

    Remarque  Si vous activez cette option, vous ne pouvez plus vous connecter à des ordinateurs exécutant des versions du système d'exploitation Windows antérieures à Windows Vista.

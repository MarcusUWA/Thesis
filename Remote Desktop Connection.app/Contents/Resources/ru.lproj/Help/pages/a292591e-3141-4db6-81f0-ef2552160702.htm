Ограничение доступа в рамках сеанса одним приложением Windows

Показать все

Пользователи, которые часто работают на сервере под управлением Windows Server с определенным приложением, могут экономить время, запуская сеанс подключения к удаленному рабочему столу только для данного приложения. Администраторы сети также могут ограничить доступ клиентских компьютеров, которые подключаются к серверу Windows, одним приложением. Если выбран этот параметр, рабочий стол Windows не отображается и недоступен.

  1. В меню RDC выберите пункт Настройки, а затем — Приложения.

  2. Установите флажок.

  3. В поле Путь и имя файла приложения введите путь и имя файла приложения, которое должно запускаться.

    Кроме того, при необходимости можно указать путь к рабочий каталогПапка, в которой установлено и выполняется приложение. Для большинства устанавливаемых приложений задан фиксированный рабочий каталог, который не требуется изменять. Некоторые устаревшие приложения, служебные программы, а также определенные скрипты и пакетные файлы рассчитаны на запуск не в той папке, в которой находится приложение. данного приложения.

Совет  Чтобы закрыть подключение, выйдите из приложения или отключитесь от компьютера под управлением Windows.

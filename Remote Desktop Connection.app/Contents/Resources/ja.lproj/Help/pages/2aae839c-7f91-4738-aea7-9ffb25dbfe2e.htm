コンピューター間でコピーおよびペーストできない

すべて表示

原因: クリップボードの一部の書式設定は貼り付けることができません。

対処方法: 互換性のあるクリップボード書式設定を備えた中間プログラムにテキストをコピーします。

Windows コンピューターからコピーして Mac に貼り付ける

  1. Windows コンピューターで、貼り付けるテキストをコピーします。

  2. Mac で TextEdit を開き、テキストを貼り付けます。

  3. TextEdit からテキストをコピーして、貼り付け先のファイルに貼り付けます。

Mac からコピーして Windows コンピューターに貼り付ける

  1. Mac で、貼り付けるテキストをコピーします。

  2. Windows コンピューターでメモ帳またはワードパッドを開き、テキストを貼り付けます。

  3. メモ帳またはワードパッドからテキストをコピーし、貼り付け先のファイルに貼り付けます。

原因: Windows コンピューターからファイルにアクセスできません。

対処方法: Remote Desktop セッションを開始する前に、Mac のディスクを Windows コンピューターに公開します。

  1. [RDC] メニューの [環境設定] をクリックします。

  2. [ドライブ] をクリックし、ポップアップ メニューで必要なオプションを選択します。

    ディスクまたはフォルダーが利用可能になり、Windows ベースのコンピューターから Mac 上のファイルにアクセスできるようになります。

    この変更を接続後に行った場合、選択したディスクまたはフォルダーは次回の接続時に公開されます。

  3. [スタート] メニューの [コンピューター] または [マイ コンピューター] をクリックします。

    利用可能になった Mac のディスクまたはフォルダーが、Windows ベースのコンピューター上の他のすべてのディスクやフォルダーと共に一覧に表示されます。

    メモ  [スタート] メニューが表示されない場合は、 [表示] メニューの [全画面表示] をクリックします。後でウインドウ モードに戻るには、画面の上端にポインターを置いて RDC のメニュー バーが表示されるのを待ち、メニュー バーが表示されたら、 [表示] メニューをクリックし、 [全画面表示] のチェックをオフにします。

  4. コピー元のディスクまたはフォルダーを開きます。

  5. [スタート] メニューの [コンピューター] または [マイ コンピューター] をクリックします。

  6. コピー先のディスクまたはフォルダーを開きます。

    これで、2 つのウインドウが開いて、コピー元の場所とコピー先の場所が表示されている状態になります。

  7. コピー元のドキュメントをコピー先の場所までドラッグします。

原因: ターミナル サービスのグループ ポリシーでドライブ共有が許可されていません。

対処方法: ターミナル サービスのグループ ポリシーの詳細については、ネットワーク管理者に問い合わせてください。

グループ ポリシーは、リモート デスクトップ接続 のどの環境設定よりも優先されます。 ターミナル サービスのグループ ポリシーでドライブ共有が禁止されているときは、コンピューター間でファイルを共有できません。

原因: ターミナル サービスのグループ ポリシーでクリップボード共有が許可されていません。

対処方法: ターミナル サービスのグループ ポリシーの詳細については、ネットワーク管理者に問い合わせてください。

グループ ポリシーは、リモート デスクトップ接続 のどの環境設定よりも優先されます。 ターミナル サービスのグループ ポリシーでクリップボード共有が禁止されているときは、コンピューター間でのデータのコピーと貼り付けはできません。
